Redakční fronta — co čeká na publikaci

Tato stránka zobrazuje články, které Claude napsal, ale které ještě nejsou publikovány v sekci Články. Každý má naplánovaný termín automatické publikace (cron běží denně v 06:00 ráno SEČ). Slouží autorovi projektu k revizi před vyjetím — můžete články pročíst, případně si poznamenat opravy předtím, než se objeví na webu.

Stránka má noindex meta tag, takže ji vyhledávače neindexují. Není ale autentizovaná — kdokoli, kdo zná URL /redakce.html, ji vidí. Pokud potřebujete přísnější ochranu, ukládejte poznámky bokem.

Články mimo automatickou publikaci

Tyto články mají v articles.json pole _review_note — typicky kvůli identifikované halucinaci nebo věcné chybě. Cron je NEpublikuje automaticky; vyžadují lidský přepis nebo rozhodnutí, zda zveřejnit.

    Jak to funguje

    Mechanismus automatické publikace

    1. Claude píše — model dostane data + zadání tématu, napíše analytický článek + zaregistruje ho do data/articles.json s published: false a scheduled_for: YYYY-MM-DD.
    2. PR + merge — článek se mergne do main branch jako nepublikovaný.
    3. Cron 06:00 — každý den ráno se spustí GitHub Actions workflow publish-articles.yml. Skript projde fronta a každý článek, jehož scheduled_for je dnes nebo dřív, přepne na published: true.
    4. Vercel rebuild — po commitu pushu cronem se Vercel automaticky rebuildne (typicky do 2 minut).
    5. Článek vychází — objeví se v /clanky.html mezi publikovanými.
