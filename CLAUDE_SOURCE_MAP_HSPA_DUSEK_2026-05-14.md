## Lokalne dostupne souhrny a orientacni struktura

- `OIS-11-05 zakladni sazby ALP`:
  - listy `VÝSTUP 2019`, `VÝSTUP 2022`, `VÝSTUP 2023`, `VÝSTUP 2024`
  - hlavicky kolem radku 18-19
  - obsahuje ICZ, nazev poskytovatele a sazby po ZP 111/201/205/207/209/211/213
- `OIS-11-17 nakladova struktura luzkove pece`:
  - list `data`, list `řádky výkazů`
  - sloupce pro skupiny nakladu/vynosu; vhodne pro 00024 / nakladove struktury jen po overeni metodiky
- `OIS-11-24 naklady ZP segmenty`:
  - list `Úhradová data – souhrn`: CPO, rok, segment, uhrada
  - list `Úhradová data – okresy`: CPO, rok, segment, kod okresu, nazev okresu, uhrada
- `OIS-11-39 referencni sit`:
  - list `Data o referenční síti`
  - DRG kod, nazev DRG, pocet HP CR, pocet HP RN, pokryti HP RN
- `OIS-11-50 PALP komplexni vystup`:
  - list `2024`
  - velmi siroky vystup: hospodareni, uhrady ZP, uvazky pracovniku, urgentni prijem, CVSP, pocty luzek, vyuziti luzek, produkce/casemix atd.

## Interni HSPA kontrakt: kam zapisovat

Novy indikator:

1. Pridat objekt do `05_M1_Starter/data/indicators.json`.
2. Vytvorit kartu `05_M1_Starter/indicators/{id}.json`.
3. V `method_card_url` pouzit `indicators/{id}.json`.
4. Pokud jsou krajska data: pridat dataset do `05_M1_Starter/data/regions.json` s `indicator_id: "{id}"`.
5. Pokud se indikator objevuje v clanku: pridat jeho ID do `linked_indicators` v `data/articles.json`.

Validace:

```bash
cd D:\Data\Dušek\hspa\05_M1_Starter
npm test
npm run validate:all
```

## Co nedelat bez rucniho overeni

- Nezakladat produkcni indikator jen z prezentacniho slidu, pokud k nemu neni dohledany primarni NZIP/XLSX zdroj.
- Neporovnavat NRHZS definici intenzivniho luzka primo s OECD ICU definici bez metodicke poznamky.
- Nepublikovat `podfinancovani_oblastni_interna_cm` jako definitivni HSPA skore, pokud Claude Code nedokaze reprodukovat vypocet z OIS-11-05 + RN nakladovosti.
- Nepridavat regiony do `regions.json`, pokud nejsou vsech 14 kraju a jasna metodika.
